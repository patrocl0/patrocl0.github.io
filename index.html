Objetivo

Mi objetivo fue desarrollar un sitio web responsivo partiendo de un diseño en Figma, asegurándome de que funcionara correctamente en diferentes tamaños de pantalla (1280px, 768px y 320px), siguiendo buenas prácticas de maquetación y control de versiones.

El principal reto fue adaptar un diseño estático a un sitio web completamente responsivo, evitando el uso excesivo de valores fijos en CSS y asegurando que el contenido se ajustara correctamente sin generar desplazamiento horizontal en distintos dispositivos y navegadores.

Características del proyecto

  • Diseño totalmente responsivo adaptado a resoluciones de 1280px, 768px y 320px.
  • Maquetación basada en Flexbox y CSS Grid para una estructura flexible y escalable.
  • Implementación de media queries para transiciones suaves entre puntos de ruptura.
  • Desarrollo siguiendo el diseño original de Figma, respetando espaciados, tipografías y layout.
  • Uso de enlaces de marcador de posición (por ejemplo, para el botón de compra de NFT).
  • Control de versiones con Git, utilizando ramas para nuevas secciones y funcionalidades.
  • Despliegue del proyecto en GitHub Pages.

¿Qué aprendí y qué logré?

Aprendí a trabajar de forma estructurada dividiendo el proyecto en etapas pequeñas, a usar Flexbox y CSS Grid para crear diseños responsivos, y a aplicar media queries de manera eficiente. También reforcé el uso de Git y ramas para organizar el desarrollo y el despliegue del proyecto en GitHub Pages, logrando un sitio funcional, adaptable y alineado con el diseño original de Figma.

Objetivo

Este proyecto es una página web desarrollada con fines educativos y prácticos, cuyo objetivo fue profundizar en la interactividad con JavaScript, el manejo del DOM, y la organización del código frontend. A partir de un diseño proporcionado en Figma, se construyó una aplicación interactiva donde los usuarios pueden visualizar, agregar y eliminar tarjetas con imágenes. Inicialmente, la aplicación carga seis tarjetas de forma dinámica utilizando JavaScript. Los usuarios pueden añadir nuevas tarjetas mediante un formulario emergente, interactuar con ellas a través del botón de “me gusta” y eliminarlas cuando lo deseen. Además, al hacer clic en una imagen, se abre una ventana emergente que muestra la imagen en mayor tamaño, respetando un diseño adaptativo y sin generar desplazamiento horizontal. En una etapa posterior, el proyecto se centró en la refactorización del código, aplicando programación orientada a objetos y JavaScript modular. Se crearon clases como Card, encargada de generar y gestionar las tarjetas, y FormValidator, responsable de validar los formularios. El código fue dividido en módulos ES6, separando responsabilidades y mejorando la mantenibilidad y escalabilidad del proyecto. Este proyecto representa la transición desde los fundamentos de JavaScript hacia una arquitectura más estructurada, aplicando buenas prácticas de desarrollo frontend, diseño responsivo y control de versiones, sentando las bases para futuras integraciones con backend.

Skills y tecnologías utilizadas en el proyecto

  • HTML5 semántico
  • CSS3 moderno
  • Diseño responsivo (Responsive Web Design)
  • Metodología BEM
  • Flexbox y CSS Grid
  • Media Queries
  • JavaScript (ES6+)
  • Manipulación del DOM
  • Manejo de eventos
  • Programación orientada a objetos (POO)
  • Validación de formularios

¿Qué aprendi o qué logre?

Aprendí a trabajar con JavaScript de forma más estructurada, creando clases como Card y FormValidator para encapsular la lógica de la aplicación. También reforcé el uso de módulos ES6, la validación de formularios, el manejo de eventos y ventanas emergentes, y la construcción de interfaces responsivas. Como resultado, logré un proyecto funcional, organizado y escalable, con una base sólida para futuras integraciones con backend.
